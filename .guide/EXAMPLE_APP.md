# Example App Requirements (Shadcn-Style)

The example app must feel like https://ui.shadcn.com in layout, navigation, and documentation clarity.

## Layout
- Left sidebar navigation with categories and search.
- Main content area with component documentation.
- Header with global search and command palette.

## Component Pages
Each component page must include:
- Title and short description
- Usage examples
- Variants and states
- Props table (if applicable)
- Code snippets with copy button

## Documentation-First UX
- Prioritize clarity and scannability.
- Keep typography and spacing consistent.
- Provide quick navigation between components and categories.
