# Getting Started

`@hieupth/reui` is a headless React UI primitive library — composition over
inheritance. Components ship styling-agnostic (no CSS framework baked in) so you
can render them with Tailwind, CSS Modules, or plain CSS.

## Installation

Install the library from npm:

```bash
npm install @hieupth/reui
```

or with pnpm / yarn / bun:

```bash
pnpm add @hieupth/reui
yarn add @hieupth/reui
bun add @hieupth/reui
```

## Peer dependencies

Most components are pure React and need no extra packages. `react-hook-form`
(the engine behind `Form` / `useForm`) ships bundled as a regular dependency,
so it is already in your tree — no separate install required. One opt-in
feature pulls in an optional peer dependency:

| Feature           | Peer dependency  | Optional? | Install                       |
| ----------------- | ---------------- | --------- | ----------------------------- |
| Motion components | `framer-motion`  | yes       | `npm install framer-motion`   |
| `Form` validation | `react-hook-form` | no — bundled | _(already installed)_      |

If you use any Motion component (`FadeInOut`, `SlideIn`, `RevealOnScroll`, …),
install `framer-motion` yourself; it is marked optional so non-motion bundles
stay lean. Tree-shaking keeps everything you don't use out of your bundle.

## Requirements

- **React 19+** (uses the new JSX transform and concurrent features).
- **Node 18+** (matches the React 19 toolchain and modern bundlers).
- A bundler that supports ESM (Vite, Next.js 14+, webpack 5, esbuild, etc.).

## Quick example

A `Button` is the simplest place to start — it is fully headless and accepts
the same props as a native `<button>`:

```tsx
import { Button } from '@hieupth/reui';

export default function App() {
  return (
    <Button onPress={() => alert('Hello from reui!')}>
      Click me
    </Button>
  );
}
```

## What's included

The library is organized into categories. Browse them from the sidebar:

- **Buttons** — `Button`, `ButtonGroup`, `Toggle`
- **Forms** — `Input`, `Select`, `Checkbox`, `Form`, `FileUpload`, …
- **Feedback** — `Dialog`, `Toast`, `Tooltip`, `Progress`, …
- **Navigation** — `Tabs`, `Menu`, `Accordion`, `Command`, …
- **Data Display** — `Table`, `DataGrid`, `Card`, `Carousel`, …
- **Layout/System** — `AspectRatio`, `Portal`, `VisuallyHidden`, …
- **Motion** — `FadeInOut`, `SlideIn`, `RevealOnScroll`, …

Every component is headless: it handles behavior, accessibility, and state, and
leaves the markup styling to you.
